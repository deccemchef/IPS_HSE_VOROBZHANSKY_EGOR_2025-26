Режимы ценообразования

Uniform Единая цена

Фирма устанавливает одну цену p для всех. Покупатель купит столько единиц, сколько ему выгодно при этой цене.

Параметр: p — цена за единицу
T(q) = p · q
Простейший режим. Часть покупателей, чья ценность ниже p, не купят ничего → возникает DWL.
PD1 Первая степень

Фирма знает точную готовность платить каждого и назначает индивидуальную цену за каждую единицу. Весь потребительский излишек уходит фирме.

Параметры: не нужны
T(q) = V(q) — вся ценность
Максимальный Profit, CS = 0, DWL = 0. Теоретический предел — в реальности недостижим.
PD2 Двухчастный тариф

Покупатель платит фиксированный взнос F при входе, плюс цену p за каждую единицу. Если не платит F — не покупает вообще.

Параметры: F — взнос, p — цена за единицу
T(q) = F + p · q, если q > 0
Через F фирма изымает часть CS. Чем меньше p — тем больше покупают, но нужен баланс с F.
PD3 Третья степень

Фирма делит покупателей на сегменты A, B, C и назначает каждому сегменту свою цену. Покупатели из разных сегментов платят по-разному за одно и то же.

Параметры: pA, pB, pC — цены по сегментам
T(q) = p[сегмент] · q
Сегмент с меньшей эластичностью — выше цена. Позволяет извлечь больше, чем Uniform.

Что значат метрики

Q
Суммарный объём покупок — сколько единиц купили все покупатели у этой фирмы.
Revenue
Выручка — сумма всех платежей покупателей. Revenue = Σ T(q).
Profit
Прибыль фирмы после вычета переменных издержек. Profit = Revenue − MC · Q.
CS
Consumer Surplus — потребительский излишек. То, что покупатели получили сверх уплаченного: CS = Σ [V(q) − T(q)].
PS
Producer Surplus — производительный излишек. В этой модели совпадает с Profit.
W
Welfare — общественное благосостояние. W = CS + PS. Показывает суммарную выгоду всех участников рынка.
W_eff
Эффективное благосостояние при цене P = MC (конкурентный рынок). Это теоретический максимум W — эталон для сравнения.
DWL
Deadweight Loss — потери благосостояния. DWL = W_eff − W. Чем больше DWL, тем менее эффективна стратегия для общества.

1. Монополия

Монополия — рыночная структура, при которой товар предлагает один продавец без близких конкурентов. Монополист сталкивается со всем рыночным спросом.

Монополист выбирает объём по условию MR = MC. По сравнению с конкурентным рынком: цена выше, объём меньше, возникают потери благосостояния DWL.

График базовой монополии

2. Основные обозначения

MB(k) — предельная полезность k-й единицы товара.

V(q) — суммарная ценность q единиц:

V(q) = MB(1) + MB(2) + ... + MB(q)

T(q) — общий платёж покупателя за q единиц.

U(q) — чистая полезность покупателя:

U(q) = V(q) − T(q)

3. Выбор покупателя

Покупатель перебирает варианты q = 0, 1, 2, ... и выбирает тот объём, который максимизирует чистую полезность.

q* = argmax U(q) = argmax [V(q) − T(q)]

4. Ценовая дискриминация

Ценовая дискриминация — продажа одного товара разным покупателям по разным ценам, не объясняемым разницей в издержках.

Условия: рыночная власть фирмы, возможность различать покупателей, ограничение перепродажи между группами.

4.1. Ценовая дискриминация первой степени

Фирма знает точную готовность платить каждого покупателя за каждую единицу и назначает индивидуальную цену. Потребительский излишек полностью переходит к фирме.

T(q) = V(q)

Фирма продаёт до Q*, где цена (MB) сравнивается с MC. Объём продаж = конкурентному — потерь благосостояния нет.

График PD1

4.2. Ценовая дискриминация второй степени

Фирма предлагает двухчастный тариф: фиксированный взнос F плюс цена за единицу p.

T(0) = 0    T(q > 0) = F + p · q

Через p фирма влияет на объём покупки, через F — изымает часть потребительского излишка.

График PD2

4.3. Ценовая дискриминация третьей степени

Фирма делит покупателей на наблюдаемые сегменты и назначает каждому сегменту свою цену.

T(q) = p[segment] · q

Сегмент с менее эластичным спросом получает более высокую цену. В модели — сегменты A, B, C.

График PD3 — три сегмента

5. Экономические показатели

Revenue — выручка фирмы: сумма всех платежей.

VarCost — переменные издержки: mc · Q.

Profit — прибыль:

Profit = Revenue − VarCost

CS — consumer surplus, потребительский излишек.

PS — producer surplus, производительный излишек (= Profit в этой модели).

W — общественное благосостояние:

W = CS + PS

W_eff — эффективное благосостояние при конкурентной цене P = MC.

DWL — потери благосостояния:

DWL = W_eff − W

6. Связь теории с симулятором

Симулятор переводит микроэкономические идеи в вычислимую форму. Для каждого покупателя задаётся функция спроса, строится функция платежа T(q), покупатель выбирает объём, максимизируя полезность.

Индивидуальные решения агрегируются в показатели фирмы и рынка: Q, Revenue, Profit, CS, PS, W, DWL.

Раздел «Сравнение режимов» позволяет запустить все четыре режима на одном наборе покупателей и увидеть, как меняются CS, PS, W и DWL при разных стратегиях ценообразования.

Раздел «Готовые сценарии» содержит предварительно настроенные примеры, соответствующие типичным задачам из учебников по микроэкономике.
