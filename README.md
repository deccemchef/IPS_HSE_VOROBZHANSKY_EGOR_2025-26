# Monopolist Strategy Studio

Интерактивный симулятор «монополии с ценовой дискриминацией». Пользователь наблюдает, как монополист (завод/платформа) взаимодействует с гетерогенными покупателями, выбирая правила ценообразования: обычная монополия с единой ценой, дискриминация I, II, III степени (включая двухчастный тариф, блок-тарифы и др.). Модель показывает динамику выручки, прибыли, потребительского/производительного излишков и потерь благосостояния при разных структурах спроса и издержек — в «живом» агентном мире (человечки ходят, товары производятся/покупаются), с привязкой к теории.


# Образовательный модуль
В приложении есть отдельный блок теории по монополии и ценовой дискриминации (ключевые определения, графики MR=MC, индикаторы эффективности, примеры тарифов, задачи и краткие конспекты). Он делает симуляцию наглядной и удобной для использования на семинарах: преподаватель выбирает сценарий, а студенты видят, как теоретические эффекты проявляются в модели в реальном времени, что помогает лучше усвоить тему.

# Модель
Пользователь управляет сценарием:
- назначает, какие покупатели пойдут в Factory, какие — в Agency (или останутся),
- переключает режим ценообразования для каждой фирмы: **Uniform / PD1 / PD2 / PD3**,
- смотрит, как у одних и тех же покупателей меняются: **количество покупки, оплата, CS, прибыль, DWL**.

## Покупатели и спрос 
У каждого покупателя для каждого товара хранится:
- `targetStock` — сколько единиц ему “нормально” иметь,
- `stock` — сколько есть сейчас,
- `decayPerDay` — сколько единиц “уходит” за день.

Определяем дефицит:
- `gap = max(0, targetStock - stock)`

Покупка в день ограничена:
- `q ∈ {0, 1, ..., gap}`

## Готовность платить за количество
Для каждого покупателя и товара есть параметры ценности:
- `a` — ценность первой единицы,
- `b` — падение ценности на каждую следующую.

Ценность k-й единицы:
- `MB(k) = max(a - b*(k-1), 0)`

Ценность q единиц:
- `V(q) = MB(1) + ... + MB(q)`

## Режимы дискриминации
Каждый режим — это функция оплаты `T(q)` (сколько платит покупатель за q единиц):

1) **Uniform**: `T(q) = p*q`
2) **PD3 (по сегментам A/B/C)**: `T(q) = p[segment]*q`
3) **PD2 (двухчастный тариф)**: `T(0)=0`, `T(q>0)=F + p*q`
4) **PD1 (персональная)**: `T(q) = V(q)` 

Параметры `p`, `F`, `p[A/B/C]` задаются **сценарием**, чтобы сравнение режимов было честным.

## Что происходит, когда покупатель пришёл
Когда покупатель приходит к выбранной фирме, система берёт его данные (`money`, `gap`, `a`, `b`, `segment`) и автоматически считает:
- сколько он купит `q*`,
- сколько заплатит `T(q*)`,
- какой получится `CS = V(q*) - T(q*)`.

Количество `q*` выбирается как лучшее по полезности:
- `U(q)=V(q)-T(q)` при ограничениях `q<=gap` и `T(q)<=money`.

## Фирмы и экономика
Для каждой фирмы:
- предельные издержки `mc > 0`,
- фиксированные издержки **0**,
- мощность `capacityPerDay` (сколько единиц можно продать за день).

За день:
- `Revenue` — сумма оплат,
- `VarCost = mc*Q`,
- `Profit = Revenue - VarCost`.

## Метрики, которые показывает симуляция
- `Q` — объём продаж
- `Revenue` — выручка
- `Profit` — прибыль
- `CS` — потребительский излишек (сумма `V - T`)
- `PS` — производительный излишек (`Revenue - VarCost`)
- `W = CS + PS` — благосостояние
- `DWL` — потери эффективности
